# Промт для Cursor: Программа поиска текста в файлах (C + Windows GUI)

## 🎯 Основное задание

Создать полнофункциональное **Windows-приложение для поиска текста в файлах** на языке **C** с графическим интерфейсом (Win32 API). По завершении работы создать скомпилированный **.exe файл**, готовый к запуску на любом Windows ПК без дополнительных зависимостей.

---

## 📋 Технические требования

### Язык программирования и компилятор
- **Язык**: C (стандарт C99 или C11)
- **Компилятор**: MinGW-w64 (GCC для Windows)
- **GUI**: Win32 API (нативный Windows API)
- **Кодировка**: UTF-8 с поддержкой кириллицы
- **Платформа**: Windows 7/8/10/11 (32/64-bit)

### Основные библиотеки
```c
#include <windows.h>      // Win32 API
#include <commctrl.h>     // Common Controls
#include <stdio.h>        // Стандартный ввод-вывод
#include <stdlib.h>       // Управление памятью
#include <string.h>       // Работа со строками
#include <dirent.h>       // Работа с директориями (MinGW)
#include <sys/stat.h>     // Информация о файлах
```

## 🔨 Компиляция и сборка

### Инструкции для MinGW (GCC)

**compile.bat** — автоматическая компиляция и сборка:

```batch
@echo off
echo ======================================
echo Компиляция File Search Tool
echo ======================================

REM Проверка наличия MinGW
where gcc >nul 2>nul
if %ERRORLEVEL% NEQ 0 (
    echo ОШИБКА: MinGW не найден в PATH!
    echo Установите MinGW-w64 и добавьте в PATH
    pause
    exit /b 1
)

echo.
echo [1/4] Компиляция ресурсов...
windres resources\resource.rc -O coff -o build\resource.res

echo [2/4] Компиляция utils.c...
gcc -c src\utils.c -o build\utils.o -Wall -O2

echo [3/4] Компиляция search.c...
gcc -c src\search.c -o build\search.o -Wall -O2

echo [4/4] Компиляция gui.c и main.c...
gcc -c src\gui.c -o build\gui.o -Wall -O2
gcc -c src\main.c -o build\main.o -Wall -O2

echo.
echo [Линковка] Создание исполняемого файла...
gcc build\main.o build\gui.o build\search.o build\utils.o build\resource.res ^
    -o build\FileSearchTool.exe ^
    -mwindows -lcomctl32 -lgdi32 -lcomdlg32 -lshlwapi

if %ERRORLEVEL% EQU 0 (
    echo.
    echo ======================================
    echo ✓ УСПЕШНО!
    echo ======================================
    echo Исполняемый файл: build\FileSearchTool.exe
    echo.
    echo Запустить программу? (Y/N)
    set /p choice=
    if /i "%choice%"=="Y" start build\FileSearchTool.exe
) else (
    echo.
    echo ======================================
    echo ✗ ОШИБКА КОМПИЛЯЦИИ!
    echo ======================================
)

pause
```

### Команда компиляции (одной строкой)

```bash
gcc src/main.c src/gui.c src/search.c src/utils.c -o build/FileSearchTool.exe -mwindows -lcomctl32 -lgdi32 -lcomdlg32 -lshlwapi -Wall -O2
```

### Флаги компиляции

- `-mwindows` — создать Windows GUI приложение (без консоли)
- `-lcomctl32` — библиотека Common Controls
- `-lgdi32` — библиотека GDI (графика)
- `-lcomdlg32` — библиотека Common Dialogs (диалоги выбора файлов/папок)
- `-lshlwapi` — библиотека Shell API
- `-Wall` — все предупреждения
- `-O2` — оптимизация уровня 2

## 🛡️ Обработка ошибок

### Типы ошибок

1. **Ошибки ввода**
   - Пустой путь поиска → MessageBox с предупреждением
   - Несуществующая директория → MessageBox с ошибкой
   - Нет ключевых слов → MessageBox с предупреждением

2. **Ошибки файловой системы**
   - Нет доступа к директории → Пропустить и записать в лог
   - Ошибка чтения файла → Пропустить файл
   - Слишком большой файл → Опционально пропустить

3. **Системные ошибки**
   - Нехватка памяти → Корректное завершение
   - Ошибка создания окна → MessageBox и выход

### Логирование

```c
// Простое логирование в файл
void logError(const char* message) {
    FILE* logFile = fopen("search_errors.log", "a");
    if (logFile) {
        time_t now = time(NULL);
        fprintf(logFile, "[%s] %s\n", ctime(&now), message);
        fclose(logFile);
    }
}
```
